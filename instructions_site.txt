INSTRUCTIONS POUR LA GESTION DU SITE C2R
=====================================

1. INFORMATIONS IMPORTANTES
-------------------------
URL du site : https://ryjits.github.io/C2R/
Dépôt GitHub : https://github.com/RYJITS/C2R.git
Email associé : y.Scheidegger@gmail.com

2. FICHIERS DE MISE À JOUR AUTOMATIQUE
------------------------------------
- check_status.bat : Pour vérifier les fichiers modifiés
- update_site.bat : Pour publier les modifications

3. COMMENT METTRE À JOUR LE SITE
------------------------------
Étape 1 : Faire vos modifications
- Modifiez les fichiers normalement dans votre éditeur
- Sauvegardez vos modifications

Étape 2 : Vérifier les modifications
- Double-cliquez sur check_status.bat
- Vous verrez la liste des fichiers modifiés

Étape 3 : Publier les modifications
- Double-cliquez sur update_site.bat
- Entrez un message décrivant vos modifications
- Attendez la confirmation de mise à jour

Étape 4 : Vérifier le site
- Attendez 2-3 minutes que GitHub mette à jour le site
- Visitez https://ryjits.github.io/C2R/
- Appuyez sur Ctrl+F5 pour forcer le rafraîchissement

4. STRUCTURE DES FICHIERS PRINCIPAUX
---------------------------------
index.html : Page principale du site
css/ : Dossier des styles
  ├── logo.css
  ├── opening.css
  ├── sidebar-icons.css
  ├── sidebar-right.css
  └── shared/
      ├── animations.css
      └── reset.css
js/ : Dossier des scripts
  ├── logo.js
  ├── opening.js
  ├── sidebar-icons.js
  ├── sidebar-right.js
  └── vibration.js

5. EN CAS DE PROBLÈME
-------------------
Si les modifications n'apparaissent pas :
1. Attendez quelques minutes
2. Forcez le rafraîchissement (Ctrl+F5)
3. Videz le cache du navigateur
4. Essayez en navigation privée

Si une erreur survient lors de la mise à jour :
1. Notez le message d'erreur
2. Vérifiez que tous les fichiers sont sauvegardés
3. Réessayez la mise à jour

6. COMMANDES GIT MANUELLES (si besoin)
-----------------------------------
cd d:/WEBSITE/C2R
git status                    : Voir les modifications
git add .                     : Ajouter les modifications
git commit -m "message"       : Enregistrer les modifications
git push                      : Publier les modifications

7. LIENS UTILES
-------------
Documentation GitHub : https://docs.github.com/fr
GitHub Pages : https://pages.github.com/
Aide Git : https://git-scm.com/doc

8. CONSEILS DE SÉCURITÉ
---------------------
- Ne jamais partager vos identifiants GitHub
- Faire des sauvegardes régulières
- Toujours vérifier les modifications avant de les publier
- Garder une copie locale du site

9. RAPPEL DES FICHIERS À NE PAS MODIFIER
-------------------------------------
- .git/ (dossier)
- .gitignore
- Les fichiers de configuration GitHub

10. POUR CRÉER UNE NOUVELLE PAGE
-----------------------------
1. Créez votre fichier HTML dans le dossier principal
2. Ajoutez les liens CSS nécessaires
3. Ajoutez les scripts JS nécessaires
4. Testez localement
5. Publiez avec update_site.bat
